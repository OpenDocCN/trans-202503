B SDDL SID ALIAS MAPPING

Chapter 5 introduced the Security Descriptor Definition Language (SDDL) format for expressing a security descriptor as a string and gave some examples of the two-character aliases that Windows supports for well-known SDDL SIDs. While Microsoft documents the SDDL format for SIDs, it provides no single resource listing all the short SID alias strings. The only available resource is the sddl.h header in the Windows SDK. This header defines the Windows APIs a programmer can use to manipulate SDDL format strings and provides a list of short SID alias strings.

Table B-1 contains the short aliases along with the names and full SIDs that they represent. The table was extracted from the header provided with the SDK for Windows 11 (OS build 22621), which should be the canonical list at the time of writing. Note that some SID aliases work only if youâ€™re connected to a domain network. You can identify these by the <DOMAIN> placeholder in the SID name, which you should replace with the name of the domain the system is connected to. Also replace the <DOMAIN> placeholder in the SDDL SID string with the unique domain SID.
