Appendix
Project Creation Checklist

Native C Project

For programs that run on the PC, start the STM32 Workbench, select a workspace if needed, and then complete the following steps:

  1. In the main window, select File▶New▶C/C++ Project.
  2. In the C/C++ Project window, select C Managed Build and click Next.
  3. In the C Project window:
    1. Fill in the name (no spaces or funny characters).
    2. In the Project Type column, select Executable▶Empty Project.
    3. In the Toolchains column, select the one for your native system. For example, if you are on Windows with Visual C++, you would choose Visual C++. If you’re on Linux, you would select the Linux C compiler, GNU gcc.
    4. Click Next.
  4. In the Configurations window:
    1. Uncheck Release.
    2. Debug should remain checked.
    3. Click Finish.
  5. In the main window:
    1. In the Project Explorer (left column), select the project.
    2. Select File▶New▶Source File.
  6. In the Source File window:
    1. In the Source File field, enter the name of your program file (ends with .c).
    2. Click Finish.
  7. Back in the main window (with the file shown in the editing pane):
    1. Enter the program.
    2. Select File▶Save All.
    3. Select Project▶Build Project.
    4. Select Run▶Run Configuration.
  8. In the Create, Manage, and Run Configurations window:
    1. In the left column, select C/C++ Application.
    2. Click New.
    3. Under C/C++ Application, click Browse.
  9. In the File dialog:
    1. Go to your workspace and open the project folder.
    2. In the project folder, open the Debug directory.
    3. Select the executable for your program.
    4. Click OK.
  10. Back in the Create, Manage, and Run Configurations window:
    1. Click Apply.
    2. Click Close.

You can now run and debug your program.

STM32 Workbench Embedded Project

For programs that run on the Nucleo board, start the STM32 Workbench, select the workspace if needed, and then complete the following steps:

  1. In the main window, select File▶New▶C Project.
  2. In the C Project window:
    1. Fill in the name (no spaces or funny characters).
    2. In the Project Type column, select Executable▶Empty Project.
    3. In the Toolchains column, select Ac6 STM32 MCU GCC.
    4. Click Next.
  3. In the Configurations window:
    1. Uncheck Release.
    2. Debug should remain checked.
    3. Click Finish.
  4. In the Target Configuration window:
    1. For the series, select STM32F0.
    2. For the board, select NUCLEO-F030R8.
    3. Click Next.
  5. In the Project Firmware Configuration window:
    1. Click Hardware Abstraction Layer (Cube HAL).
    2. Download the target firmware (needed only once).
    3. Click Finish.
  6. Back in the main window:
    1. In the Project Explorer (left column), select the project.
    2. Select Project▶Properties.
  7. In the Properties window:
    1. Expand C++ Build.
    2. Go to Settings.
    3. Go to the Tool Settings tab.
    4. Select MCU GCC Compiler▶Debugging.
    5. Debug level: Maximum (-g3).
    6. Select MCU GCC Compiler▶Miscellaneous.
    7. Other flags: -fmessage-length=0 0Wa,adhls=$(@:%.o=%.lst).
    8. Turn on Listings.
    9. Click Apply.
    10. Click OK.
  8. In the main window:
    1. In the Project Explorer (left column), select the project.
    2. Expand the project.
    3. Expand the src directory.
    4. Click main.c to edit it.
    5. Enter the program.
    6. Select File▶Save All.
    7. Select Project▶Build Project.
    8. Select Project▶Run or Project▶Debug.
